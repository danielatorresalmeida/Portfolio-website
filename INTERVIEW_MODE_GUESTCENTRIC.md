# Interview Mode: Guestcentric

## Top 3 Projects to Present
1. NextPath Angular Dashboard (featured case study)
2. Responsive Portfolio Website
3. API QA Test Suite

## 60-Second Pitch per Project
1. NextPath Angular Dashboard
I built an Angular dashboard concept to organize developers, projects, and Agile tasks more clearly. The main problem was scannability: users needed to understand priorities faster. I focused on hierarchy, layout grouping, and reusable component structure so design decisions map directly to frontend implementation. It is still in progress, and my next step is formalizing spacing, typography, and component rules into a stronger Figma design system.

2. Responsive Portfolio Website
I designed and implemented this portfolio with HTML, CSS, and JavaScript to improve clarity and recruiter usability on desktop and mobile. I focused on section hierarchy, clean CTAs, and readable structure, then validated behavior with manual checks and Selenium smoke tests. It shows how I work across design and code, not only mockups.

3. API QA Test Suite
I built a Python + Pytest suite for API checks (status, schema, and business rules) with readable report output. While it is not a UI project, it demonstrates structured thinking, quality ownership, and release discipline that also improves frontend delivery confidence.

## Strongest Homepage/About Sentence
"I design clean, responsive web experiences in Figma and bring them closer to production with HTML, CSS, and frontend thinking."

## Weak Points Interviewers May Challenge
1. Junior level and limited years of direct web design role experience.
2. Some projects are still in progress, especially NextPath.
3. Mixed profile (UI/UX, frontend, QA, software) may look broad.

## Honest Response Strategy
1. Junior level
Be direct: junior level is accurate, and the value is practical execution, fast learning, and reliable delivery habits.

2. In-progress work
Show current artifacts and explain concrete next steps already identified (design system refinement, component rules, and validation workflow).

3. Broad profile
Frame breadth as useful for Guestcentric: design decisions are made with browser implementation and usability in mind, not isolated visual mockups.
